## 초기설정(프로젝트 처음 시작할 때만)
### 1. 이 폴더를 Git이 관리하도록 설정
    git init   
    의미: "이제부터 이 폴더 안의 변화를 모두 기록해줘."
   
### 2. 원격 저장소(GitHub) 연결
    git remote add origin https://github.com/사용자명/저장소명.git   
    의미: "이 프로젝트의 서버 주소는 여기야."

## 새 브랜치 만들고 동시에 이동
    git checkout -b 내-브랜치-이름   
    의미: "기존 코드(보통 main)를 복사해서 '내-브랜치'라는 새 방을 만들고 바로 글로 갈게."

<br />
<br />

## 다른 브랜치에서 내 브랜치로 이동
git checkout 내-브랜치-이름

<br />
<br />

## 내 브랜치에 수정한 파일 업로드
### 1. 수정된 파일들을 '스테이징 영역(장바구니)'에 담기
git add .   
의미: "현재 폴더의 모든 변경 사항을 저장 대기 목록에 올릴게."

### 2. 버전 기록(커밋) 만들기
git commit -m "무슨 작업을 했는지 설명"   
의미: "이 작업물에 이름을 붙여서 내 컴퓨터에 확정 저장할게."

### 3. 서버(GitHub)로 전송
git push origin 내-브랜치-이름   
의미: "내 컴퓨터에만 있는 이 기록들을 서버(origin)의 '내-브랜치'로 보낼게."



## 다른 브랜치(예 : main) 파일을 내 브랜치로 가져오기
(전제: 현재 내 브랜치에 접속 중인 상태여야 함)

### 1. 서버의 최신 정보 업데이트 (한 번 해주는 게 좋음)
git fetch origin

### 2. 서버의 main 브랜치 내용을 내 브랜치로 당겨와서 합치기
git pull origin main
